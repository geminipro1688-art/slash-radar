五大數據維度，怎麼變成分數

評分完全透明、可解釋——不是黑箱。下面是每個維度的中立定義。
OI 結構（權重最高）

未平倉量與價格同向上升＝資金主動增倉（增倉上行）；價跌但 OI 增＝空方增倉；價漲 OI 減＝空頭回補的輕倉反彈。

價格動能（1h）

近 1 小時漲跌幅，衡量短線動能方向與強弱。

資金費率

正費率＝多頭付費、多方擁擠；負費率＝空頭擁擠。極端值會標記「費率背離」。

主動買賣量 CVD

主動買量減主動賣量。價漲但 CVD 轉弱＝動能衰減；價跌但買盤增＝下方吸籌。

多空比

多空帳戶比，過度擁擠時做反向風險提示（多方／空方擁擠）。

槓桿擁擠（風險標籤）

OI／市值比偏高時標記「槓桿擁擠」，只示警、不計分方向。

常見問題

這個評分代表可以買進或做空嗎？

不代表。評分只是對「已發生合約數據」的統計描述，衡量數據偏多方或偏空方結構的強度，不是、也不應被解讀為進場、止損、止盈或買賣指令。每張卡都附免責聲明。這也是我們和「直接給買賣點」的工具最大的不同。

資料來源是什麼？準嗎？

全部來自 OKX、幣安的公開合約 API 與 CoinGecko 市值資料。OI 與資金費率做雙所聚合（USD 計價、缺值跳過不污染），1h 動能有 K 線備援，盡量讓每個維度的數據完整可信，並在卡上標出「資料完整度」。
